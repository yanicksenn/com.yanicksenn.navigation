# Constrained Rotation Agent Strategy Implementation Plan

## Objective
Implement a new navigation strategy (`ConstrainedRotationAgentStrategy`) and its definition (`ConstrainedRotationAgentDefinition`) that constrains the agent to a constant forward speed and a maximum rotation angle per second. The agent must strictly remain within the `WalkableCubes` of the NavMesh during its movement, even if this means it has to take a wider, "larger path" to accommodate its turning radius.

## Analysis & Complexity
A standard A* pathfinder on a grid or octree generates piecewise linear paths. For an agent with non-holonomic constraints (cannot instantly turn, constant forward speed), following these sharp linear paths directly is impossible without overshooting. The turning radius is defined by `R = Speed / MaxRotationSpeed`. 

If a corner is too sharp, the agent's turn will naturally swing it outside the walkable area. To prevent this, the agent must either:
1. **Swing wide before the turn:** Steer towards the outer edge of the walkable corridor before the corner to maximize available space.
2. **Compute a Kinematically Valid Path:** Use curves (like Dubins curves) that fit within the bounds of the provided path cubes.

Since rewriting the global `NavMeshPathfinder` to be a full Hybrid A* (which searches in 3D state-space: X, Y, Heading) is out of scope for a single agent strategy, we will handle this via a **Local Trajectory/Corridor Planner** inside the new `ConstrainedRotationAgentStrategy`.

## Proposed Architecture

### 1. `ConstrainedRotationAgentDefinition`
A ScriptableObject inheriting from `NavAgentDefinition`.
- **Fields:**
  - `speed` (float): The constant forward speed (units per second).
  - `maxRotationSpeed` (float): The maximum rotation speed (degrees per second).
- **Methods:**
  - `CreateStrategy(NavMeshAgent agent)`: Returns a new instance of `ConstrainedRotationAgentStrategy`.

### 2. `ConstrainedRotationAgentStrategy`
Inherits from `NavAgentStrategy`.
- **State:**
  - `Vector3 currentVelocity`: Keeps track of the agent's current forward heading.
  - `Vector3[] basePath`: The original path generated by `NavMeshPathfinder`.
  - `Bounds[] corridor`: The cubes (`CurrentPathCubes`) forming the safe walkable space.
  - `Vector3 currentSteeringTarget`: The dynamic point the agent is aiming for.

- **Local Corridor Navigation Algorithm:**
  Instead of blindly steering towards the next waypoint, the strategy will continuously calculate valid circular arcs based on its current heading and the constraints.
  
  **Step-by-step per frame (`Update`):**
  1. **Identify the Corridor:** We use the `CurrentPathCubes` provided by the pathfinder as our safe bounds.
  2. **Lookahead Point (Pure Pursuit):** Find a point along the `basePath` at a lookahead distance `L` (proportional to speed and turning radius).
  3. **Calculate Turning Arc:** Determine the required angular velocity to reach the lookahead point, clamped by `maxRotationSpeed`.
  4. **Safety Verification:** Project the movement arc forward (e.g., sample the next $N$ frames of movement). Check if these future positions fall outside the `corridor`.
  5. **Dynamic Evasion (The "Larger Path"):** If the projected arc exits the corridor (e.g., hitting an inner corner):
     - The strategy dynamically offsets its steering target towards the *opposite* side of the corridor to swing wide.
     - We can achieve this by casting rays outward from the collision point against the corridor bounds and picking a temporary target that forces the agent to widen its arc.
  6. **Apply Movement:** Rotate the agent's forward vector by the clamped angular velocity, then move forward by `speed * deltaTime`.

### 3. Verification & Bounds Checking
To verify if a point is within the walkable area efficiently, we will write a helper method in the strategy:
- `bool IsPointInCorridor(Vector3 point, Bounds[] corridor)`: Iterates through the local `CurrentPathCubes` and checks if `Bounds.Contains` is true for any cube. This is extremely fast since the path usually consists of only a handful of cubes.
